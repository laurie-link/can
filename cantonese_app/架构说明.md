# 🏗️ 应用架构说明

## 📱 混合架构设计

应用采用**混合架构**，既支持完全独立运行，也支持使用本地后端服务器以获得更好的功能。

### 🎯 功能模块

#### 1. **翻译和解释** ✅ 完全独立
- **直接调用 AI API**（如 SiliconFlow、OpenAI）
- **无需后端服务器**
- 用户只需配置 API Key 和模型

#### 2. **粤拼标注** 🔄 混合模式
- **优先使用后端服务器**（推荐）
  - 使用 PyCantonese 库，准确可靠
  - 需要运行 Python 后端服务器
- **备选方案：AI 生成**
  - 如果没有后端服务器，使用 AI 生成粤拼
  - ⚠️ 准确性可能不如 PyCantonese

#### 3. **语音生成** 🔄 混合模式
- **优先使用后端服务器**（推荐）
  - 使用 edge-tts 库，质量好
  - 需要运行 Python 后端服务器
- **备选方案：系统 TTS**
  - 如果没有后端服务器，使用 Flutter TTS（系统 TTS）
  - 支持粤语的系统可以使用本地 TTS

## 🔧 配置方式

### 方案一：完全独立运行（推荐用于分发）

**配置：**
1. **AI API 配置**（必需）
   - Base URL: `https://api.siliconflow.cn/v1`
   - API Key: 您的 API 密钥
   - 模型名称: 通过"获取模型列表"选择

2. **后端服务器**（可选）
   - 留空即可
   - 粤拼和语音将使用备选方案

**优点：**
- ✅ 完全独立，不需要本地服务器
- ✅ 易于分发和安装
- ✅ 用户只需配置 API Key

**缺点：**
- ⚠️ 粤拼准确性可能不如 PyCantonese
- ⚠️ 语音质量取决于系统 TTS

### 方案二：使用后端服务器（推荐用于最佳体验）

**配置：**
1. **AI API 配置**（必需）
   - Base URL: `https://api.siliconflow.cn/v1`
   - API Key: 您的 API 密钥
   - 模型名称: 通过"获取模型列表"选择

2. **后端服务器**（推荐）
   - 后端地址: `http://192.168.x.x:6783`（您的电脑 IP）
   - 需要运行 Python 后端服务器

**优点：**
- ✅ 粤拼准确性高（使用 PyCantonese）
- ✅ 语音质量好（使用 edge-tts）
- ✅ 功能完整

**缺点：**
- ⚠️ 需要运行后端服务器
- ⚠️ 手机和电脑需要在同一 WiFi

## 🚀 后端服务器设置

### 启动后端服务器

```bash
cd backend
pip install -r requirements.txt
python main.py
```

后端将在 `http://0.0.0.0:6783` 运行。

### 获取电脑 IP 地址

**Windows:**
```bash
ipconfig
# 查找 IPv4 地址，如 192.168.1.100
```

**Mac/Linux:**
```bash
ifconfig
# 或
ip addr show
```

### 在应用中配置

设置 → 后端服务器地址 → 输入 `http://您的IP:6783`

例如：`http://192.168.1.100:6783`

## 🔍 技术细节

### 翻译和解释
- 直接调用 OpenAI 兼容的 API
