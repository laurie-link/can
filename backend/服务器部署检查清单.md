# ✅ Debian 服务器部署检查清单

## 📋 部署前准备

- [ ] 确认服务器有公网 IP 或域名
- [ ] 确认可以通过 SSH 访问服务器
- [ ] 确认有 root 或 sudo 权限
- [ ] 确认 Python 3.9+ 已安装

## 🔧 服务器配置

### 基础环境
- [ ] 更新系统：`sudo apt update && sudo apt upgrade -y`
- [ ] 安装必要软件：`sudo apt install -y python3 python3-pip python3-venv git nginx supervisor ufw`
- [ ] 配置防火墙：允许 22, 80, 443 端口

### 应用部署
- [ ] 创建应用目录：`/opt/cantonese-backend`
  ```bash
  sudo mkdir -p /opt/cantonese-backend
  sudo mkdir -p /opt/cantonese-backend/backend/static/audio
  sudo chown -R $USER:$USER /opt/cantonese-backend
  cd /opt/cantonese-backend
  ```

- [ ] 上传代码到服务器
  ```bash
  # 方法一：使用 scp 从本地电脑上传（在本地电脑执行）
  # 说明：将本地项目的 backend/ 目录下的所有内容复制到服务器的 /opt/cantonese-backend/backend/ 目录
  # 本地目录结构：cantonese/backend/ (包含 main.py, requirements.txt, models/, services/ 等)
  # 服务器目标目录：/opt/cantonese-backend/backend/
  
  # 在本地项目根目录（cantonese/）执行：
  scp -r backend/* user@your-server:/opt/cantonese-backend/backend/
  
  # 或者复制整个 backend 目录（包括目录本身）：
  scp -r backend user@your-server:/opt/cantonese-backend/
  # 这样会创建 /opt/cantonese-backend/backend/ 目录
  
  # 方法二：使用 Git（在服务器执行）
  cd /opt/cantonese-backend
  git clone <your-repo-url> .
  # 然后移动 backend 目录内容到正确位置（如果仓库结构不同）
  # 或者如果仓库根目录就是 backend，则：
  # git clone <your-repo-url> backend
  ```

  # Python 3.8: python3.8-venv

  #
  # 创建虚拟环境

### Nginx 配置
- [ ] 创建 Nginx 配置文件
  ```bash
  sudo nano /etc/nginx/sites-available/cantonese-backend
  ```
  添加以下内容（替换 `your-server-ip` 为实际 IP 或域名）：
  ```nginx
  server {
      listen 80;
      server_name your-server-ip _;

      client_max_body_size 10M;

      location / {
          proxy_pass http://127.0.0.1:6783;
          proxy_http_version 1.1;
          proxy_set_header Upgrade $http_upgrade;
          proxy_set_header Connection 'upgrade';
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
          proxy_set_header X-Forwarded-Proto $scheme;
          proxy_cache_bypass $http_upgrade;
          
          proxy_connect_timeout 60s;
          proxy_send_timeout 60s;
          proxy_read_timeout 60s;
      }

      location /static/ {
          alias /opt/cantonese-backend/backend/static/;
          expires 1h;
          add_header Cache-Control "public, immutable";
      }
  }
  ```

- [ ] 配置反向代理到 `http://127.0.0.1:6783`
  ```bash
  # 已在上面配置文件中设置 proxy_pass http://127.0.0.1:6783;
  ```

- [ ] 配置静态文件路径
  ```bash
  # 已在上面配置文件中设置 location /static/
  # 确保目录存在且有正确权限
  sudo chown -R www-data:www-data /opt/cantonese-backend/backend/static/
  ```

- [ ] 测试配置：`sudo nginx -t`
  ```bash
  sudo nginx -t
  ```

- [ ] 重启 Nginx：`sudo systemctl restart nginx`
  ```bash
  # 启用站点
  sudo ln -sf /etc/nginx/sites-available/cantonese-backend /etc/nginx/sites-enabled/
  sudo rm -f /etc/nginx/sites-enabled/default
  
  # 测试配置
  sudo nginx -t
  
  # 重启 Nginx
  sudo systemctl restart nginx
  sudo systemctl enable nginx
  ```

### Supervisor 配置
- [ ] 创建 Supervisor 配置文件
  ```bash
  sudo nano /etc/supervisor/conf.d/cantonese-backend.conf
  ```
  
  **如果使用虚拟环境**，添加以下内容：
  ```ini
  [program:cantonese-backend]
  command=/opt/cantonese-backend/venv/bin/uvicorn main:app --host 0.0.0.0 --port 6783 --workers 4
  directory=/opt/cantonese-backend/backend
  user=www-data
  autostart=true
  autorestart=true
  stderr_logfile=/var/log/cantonese-backend/error.log
  stdout_logfile=/var/log/cantonese-backend/access.log
  environment=PATH="/opt/cantonese-backend/venv/bin"
  ```
  
  **如果不使用虚拟环境**，添加以下内容：
  ```ini
  [program:cantonese-backend]
  command=/usr/bin/uvicorn main:app --host 0.0.0.0 --port 6783 --workers 4
  directory=/opt/cantonese-backend/backend
  user=www-data
  autostart=true
  autorestart=true
  stderr_logfile=/var/log/cantonese-backend/error.log
  stdout_logfile=/var/log/cantonese-backend/access.log
  environment=PATH="/usr/bin:/usr/local/bin"
  ```
  
  注意：如果不使用虚拟环境，确保 uvicorn 已安装在系统路径中（通常在 /usr/local/bin/uvicorn 或 /usr/bin/uvicorn）

- [ ] 配置日志目录和权限
  ```bash
  sudo mkdir -p /var/log/cantonese-backend
  sudo chown www-data:www-data /var/log/cantonese-backend
  ```

## 🆘 故障排查

如果遇到问题，检查：

1. **服务无法启动**
   - 查看 Supervisor 日志：`sudo supervisorctl tail cantonese-backend stderr`
   - 检查 Python 环境：`/opt/cantonese-backend/venv/bin/python --version`
   - 手动测试启动：`cd /opt/cantonese-backend/backend && source ../venv/bin/activate && python main.py`

2. **无法从外网访问**
   - 检查防火墙：`sudo ufw status`
   - 检查 Nginx：`sudo systemctl status nginx`
   - 检查端口监听：`sudo netstat -tlnp | grep 6783`

3. **SSL 证书问题**
   - 检查证书：`sudo certbot certificates`
   - 手动续期：`sudo certbot renew`
